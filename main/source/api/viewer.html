Viewer-specific actions (e.g., PREV_ENV/NEXT_ENV for native).

tick() → bool[source]#

Advance the viewer by one tick.

Physics and rendering are decoupled: physics steps are governed by a sim-time budget (accumulated from wall time * speed multiplier), while rendering always happens at the configured frame_rate. This keeps the display smooth even at slow playback speeds.

Returns True when a render frame was produced, False otherwise.
