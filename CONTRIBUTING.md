# Contributing

We welcome contributions! Bug fixes, documentation improvements, and new features are all appreciated.

## Workflow

1. Fork the repository and create a feature branch
2. Make your changes with clear commits
3. Test locally (`make test-all`)
4. Submit a pull request

## Changelog

We follow [Keep a Changelog](https://keepachangelog.com/) conventions.

Add entries to the "Upcoming version" section in `docs/source/changelog.rst` under the appropriate category (Added/Changed/Fixed).
